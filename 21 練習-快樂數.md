## 2. 快樂數-檢查是否快樂數

#### 程式名稱: hw21_2.py
``` python
(1) 建立一個[快樂數]類別
    (1-1) 類別名稱: Happy
    (1-2) 建構元: 傳入[編號](no), [待檢查數字](num) 等2個參數.
                  將參數存入實體中, 成為實體的成員.
    (1-3) [分拆數字]方法, split(self, number).
          其中的number是一個數字, 依其各個位數, 依序存在一個清單中再回傳.
          
          例如: 傳入的number是362, 請回傳[3, 6, 2]. 
          (假設[待檢查數字]中, 不會有數字之外的其他內容)   
          
    (1-4)  [檢查是否快樂數]方法, check(self).     
           [待檢查數字] 的所有數位之平方和，得到的新數再次求所有數位的平方和，如此重複進行，如果其中的平方和之值出現為1, 就是[快樂數].           
           例如: 28和32是快樂數, 原因如下:
           
           (1) 28 -> 2*2 + 8*8 = 68 
                  -> 6*6 + 8*8 = 100 
                  -> 1*1 + 0*0 + 0*0 = 1 (快樂數)
              
           (2) 32 -> 3*3 + 2*2 = 13
                  -> 1*1 + 3*3 = 10
                  -> 1*1 + 0*0 = 1 (快樂數)
                  
           如果在求所有數位的平方和的過程中, 出現過去已出現的平方和, 表示再算下去也會一直循環, 它不是快樂數.
           例如: 15和21不是快樂數, 原因如下:
           
           (1) 15 -> 1*1 + 5*5 = 26
                  -> 2*2 + 6*6 = 40 
                  -> 4*4 + 0*0 = 16
                  -> 1*1 + 6*6 = 37
                  -> 3*3 + 7*7 = 58
                  -> 5*5 + 8*8 = 89
                  -> 8*8 + 9*9 = 145
                  -> 1*1 + 4*4 + 5*5 = 42
                  -> 4*4 + 2*2 = 20
                  -> 2*2 = 4
                  -> 4*4  = 16 (與第3次平方和相同, 不是快樂數)
            
           (2) 21 -> 2*2 + 1*1 = 5
                  -> 5*5 = 25
                  -> 2*2 + 5*5 = 29
                  -> 2*2 + 9*9 = 85
                  -> 8*8 + 5*5 = 89
                  -> 8*8 + 9*9 = 145
                  -> 1*1 + 4*4 + 5*5 = 42
                  -> 4*4 + 2*2 = 20
                  -> 2*2 = 4
                  -> 4*4  = 16 
                  -> 1*1 + 6*6 = 37
                  -> 3*3 + 7*7 = 58
                  -> 5*5 + 8*8 = 89  (與第4次平方和相同, 不是快樂數)
          
(2) 產生實體
(3) 顯示資料
```

#### 測試
``` python
1, 32    
2, 97 
3, 139 
4, 362 
5, 228 
6, 399
```

#### 輸出
``` python
編號:1, 是否快樂數:True
編號:2, 是否快樂數:True
編號:3, 是否快樂數:True
編號:4, 是否快樂數:True
編號:5, 是否快樂數:False
編號:6, 是否快樂數:False
```
